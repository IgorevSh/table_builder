# Журнал работ

Веб-приложение для прораба: учёт выполненных работ на стройобъекте по дням.

## Стек и обоснование выбора


| Слой          | Технология                       | Почему                                                                                                                |
| ------------- | -------------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| Фронтенд      | **React 18 + TypeScript + Vite** | для быстрого построения SPA и типизации                                                                               |
| Бэкенд        | **NestJS**                       | Модульная структура, возможность удобного масштабирования, встроенная поддержка многих внешних бибилиотек и валидации |
| БД            | **PostgreSQL**                   | Надёжное хранение связанных данных                                                                                    |
| ORM           | **TypeORM**                      | Нативная интеграция с NestJS и декораторы для связей                                                                  |
| Валидация API | **class-validator**              | нативные правила в NestJS                                                                                             |
| UI            | **Собственные стили (CSS)**      | Для простого проекта хватит и стилей без UI и scss                                                                    |


Структура репозитория: `backend/` и `frontend/` — отдельные приложения, общение только через REST API.

## Возможности

- Список записей: дата, вид работ, объём с единицей, ФИО исполнителя
- Фильтр по периоду дат и сортировка
- Создание, редактирование и удаление записей 
- Справочник видов работ (таблица в БД, начальное наполнение через хардкод)

## Запуск

### 1. База данных

Поднять PostgreSQL: локально или удаленно

### 2. Бэкенд

#### Переменные окружения (БД)

Файл `backend/.env`:
```
DB_HOST=localhost
DB_PORT=5432
DB_USER=journal
DB_PASSWORD=journal
DB_NAME=work_journal
PORT=3001
```
При локальном запуске:

```bash
cd backend
cp .env.example .env   # если файла .env ещё нет
npm install
npm run start:dev
```

При развертывани  в контейнере/pm2:

```bash
cd backend
cp .env.example .env   # если файла .env ещё нет
npm install
npm run build
cp .env ./dist/.env
npm run start:prod
```

При первом запуске  создаются таблицы и автоматически заполняется справочник видов работ.

Описание запросо (Эндпоинты):

- `GET /work-types` — справочник видов работ
- `GET /journal-entries?dateFrom=&dateTo=&sort=asc|desc` — список записей
- `POST /journal-entries` — создание
- `PATCH /journal-entries/:id` — обновление
- `DELETE /journal-entries/:id` — удаление

### 3. Фронтенд

#### В отдельном терминале:

Для локального запуска:
```bash
cd frontend
npm install
npm run dev
```
При развертывании:
```bash
cd frontend
npm install
npm run build
npm run preview
```

Запросы к API проксируются через Vite (`/api` → `http://localhost:3001`).


При развертывании пробрасывать frontend/dist через веб-прокси (nginx например)
